# 长篇小说详细梗概生成技能

## 概述

本技能用于将超大长篇小说文件（几十MB、上千章）自动拆分并生成详细的叙述性故事梗概。核心思路是：用脚本检测章节边界 → 按固定大小分段 → 为每段创建一个子代理并行撰写梗概 → 合并去重排序输出。

## 工作流程

### 第一步：分析小说结构

运行章节检测脚本，了解小说的章节总数、行数和文件大小：

```bash
python3 <skill-path>/scripts/detect_chapters.py <小说文件路径>
```

脚本会输出：
- 章节总数和使用的匹配模式
- 自动计算的分段方案（每段约50-80章）
- 每段的起止行号

如果脚本无法识别章节模式（非标准格式），手动用 `grep` 查找章节标记：

```bash
grep -n -E '^第[一二三四五六七八九十百千万\d]+[章节回卷集部篇]' <文件> | head -20
```

### 第二步：创建输出文件

在小说同目录下创建梗概文件，写入标题头：

```
【<小说名> 详细梗概】
全书共<X>章，分<Y>段进行详细梗概。
```

### 第三步：批量创建子代理

根据分段方案，为每段创建一个子代理。每个子代理的任务模板：

```
你是一个小说梗概撰写专家。请完成以下任务：

1. 阅读小说文件 `<文件路径>` 的第 <起始行> 行到第 <结束行> 行（第<X>章到第<Y>章）
2. 撰写这些章节的**详细故事梗概**
3. 将梗概**追加**写入 `<输出文件路径>`

**梗概要求：**
- 开头写：`========== 第<N>部分：第<X>-<Y>章 ==========`，然后空行
- 逐章或按情节段落梳理，不要遗漏重要情节
- 必须包含：关键人物（首次出场时注明身份）、重要事件、情节转折、伏笔与悬念
- 用叙述性语言写成连贯的段落，不要写成简短的条目列表
- 不要一笔带过，要详细描述每个重要场景的内容和意义

用中文撰写。使用 Bash 的 cat 和追加重定向 >> 来写入文件。
```

**并发策略：** 尽量同时启动所有子代理（`run_in_background: true`）。如果遇到 429 限流错误，记录失败的分段，在当前批次完成后再重试失败的段。

### 第四步：等待完成并监控进度

子代理全部在后台运行。可以通过检查输出文件来监控进度：

```bash
# 查看已完成的分段数
grep -c '==========' <输出文件>

# 查看文件大小
du -h <输出文件>
```

### 第五步：去重、排序和清理

由于并行写入和重试，文件中可能存在重复分段。运行清理脚本：

```bash
python3 <skill-path>/scripts/clean_summary.py <梗概文件路径>
```

脚本会：去除重复分段（保留最长版本）、按章节顺序重新排列、清理多余空行。

## 梗概撰写要点

指导子代理撰写梗概时，强调以下几点：

1. **叙述性而非列表式**：用连贯的段落叙述故事，不要写成"第X章：XXX"的条目清单
2. **关键人物标注身份**：首次出场的人物必须注明其身份、背景和与主角的关系
3. **事件不一笔带过**：重要战斗、对话、决策要描述过程和结果，不能只说"XX战胜了YY"
4. **伏笔与悬念**：明确标注哪些情节是伏笔、哪些是未解之谜
5. **情节转折**：指出故事在哪里发生了重大方向变化，以及变化的原因

## 分段大小建议

| 小说规模 | 建议每段章节数 | 预期子代理数 |
|----------|--------------|------------|
| < 5MB, < 300章 | 全文不拆或2-3段 | 1-3 |
| 5-10MB, 300-1000章 | 30-50章/段 | 10-25 |
| 10-20MB, 1000-1500章 | 50-80章/段 | 20-30 |
| > 20MB | 80-100章/段 | 25-40 |

## 处理限流

大批量子代理可能触发 API 限流（429 错误）。应对策略：

1. 记录所有失败的分段（检查输出文件中缺少的分段标题）
2. 等当前批次的大部分完成后再启动重试
3. 重试时可以适当减少并发数量

## 输出格式

最终梗概文件结构：
```
【<小说名> 详细梗概】
全书共<X>章，分<Y>段进行详细梗概。

========== 第一部分：第1-50章 ==========
<详细叙述性梗概内容...>

========== 第二部分：第51-100章 ==========
<详细叙述性梗概内容...>

...依此类推
```
